## 这本书为谁而写

AI 已经能替你写代码了——但**判断 AI 写得对不对**，靠的是另一种东西：系统直觉。知道十万人同时点击时哪里先塌、知道缓存过期的那一瞬间会发生什么、知道"加一台机器"为什么可能引发雪崩。

这本书把这些直觉写成了故事。舞台在灵枢·信息大陆：小灵体们抢购月华手炉、后仓的老管家 MySQL 一次只能翻几千页账、快手灵 Redis 住在门口的高台上、渠工灵 MQ 修水库削洪峰。**组件都用真名登场**，每话结尾的「📜 现实界的记载」告诉你故事在真实世界里叫什么，「名词钩子表」让你温习时扫一眼就能拉起整话。

两类读者都照顾到了：**不写代码的人**，读完能听懂工程师在吵什么、能对 AI 生成的方案问出对的问题；**写代码的人**，会在熟悉的名词背后看到一条统一的暗线——

> **架构没有魔法，只有明码标价的取舍；所谓大局观，就是在下单前先看清账单。**

## 怎么读

- **像读小说一样读**——每话一个不断升级的危机，一根轴讲到底，不需要任何前置知识
- **想温习**：直接翻第十话结尾的**全季公理总表**，一表拉起全书
- **想较真**：每话的「📜 现实界的记载」和「名词钩子表」都是真术语、真数字，欢迎查证——本书发布前经过逐条事实核查
- **想学得更深**：读[附录](附录-读者研讨实录.md)——这本书由一场真实共读长成，附录里有真实读者的追问实录，和"如何把书锚到你自己经验上"的方法

## 这本书的写法（也是它想示范的东西）

正文由 AI 执笔，但**每一话的存在都由一位真实读者的追问推动**：第二话来自"请求到底怎么走的"这个困惑，第三话来自"给我编一个排查故事"的点播。读者的思考反过来塑造了书——这本身就是一次"人给判断、AI 给带宽"的协作示范，也是书名里"vibe coding 时代"的另一层意思。
